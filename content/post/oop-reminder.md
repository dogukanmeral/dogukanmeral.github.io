```
data before action
in OOP decompose to data types (in procedural decompose to functions)

class(data type definition)
field(data member)
method(function member)

object/instance: piece of data
encapsulation: methods act as "interface" to object fields
public(visible anywhere) VS private(visible only to methods of own type)

inheritance(a type including all members from another type)
! circular inheritance is illegal(doesn't make any sense)
is-a VS has-a relationship (think before using inheritance)
multiple inheritance (java and c# allow ONLY single inheritance)

overriding: redefining an inherited method
x.foo()
polymorphism: depends upon type of x
class member: member of class itself but not it's instances (static fields and methods in Java)
constructor: sets up an instance at it's creation
interface: shared set of methods in common
abstract class: a class which cannot be instantiated (just meant to serve as an ancestor)
overloading: having multiple methods with same name and different arguments
```
